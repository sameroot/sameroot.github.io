mysql对url的处理

发表于 2015-08-12

为了数据查询的方便，有些访问日志直接存储在mysql里面作为格式化的数据。而访问数据里面的url是一个棘手的问题，特别是url后面的参数：参数个数和名称、数值是千变万化的。一种做法是有张单独的url参数表，跟url本身是多对一的关系；另一种做法是url参数原模样存在一个字段内（uri和args还是存两个字段的好，因为处理起来问号会很麻烦）。第一种做法更合理，但大多数情况处理起来比较麻烦；第二种做法从参数元素角度考虑是违反第一范式的，但是大多数情况是简单并且合理的（因为大多数情况是计算总pv、uv或者某个uri的pv、uv等）。本文就是在讨论基于所有args存储为一个字符串的情况下，如何处理指定args的逻辑

常见场景：某个args的条件限制

常见场景：根据args里面某个值进行分组统计
